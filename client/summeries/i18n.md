# Multilingual Support (i18n) — Implementation Notes

## Overview

Full EN/HU multilingual support implemented using `react-i18next` + `i18next`.
Language is persisted in `SettingsContext` (localStorage) and synced to i18next
on mount and whenever it changes.

---

## Packages

```bash
npm install react-i18next i18next i18next-browser-languagedetector
```

---

## Architecture

### Language stored in SettingsContext

`language: "en" | "hu"` added to `SettingsState`. Defaults to `"en"` on first
load. Persisted to localStorage via the existing `useEffect`. A second
`useEffect` calls `i18n.changeLanguage(settings.language)` whenever the value
changes, keeping i18next in sync.

### Init file — `src/lib/i18n.ts`

Imports all locale JSON files and calls `i18n.use(initReactI18next).init(...)`.
Must be imported in `main.tsx` **before** any component renders.

### Translation files — `src/locales/`

```
src/locales/
  en/
    auth.json       login, signup, password reset
    nav.json        navbar items and account menu
    settings.json   settings page toggles and language picker
    profile.json    profile page, UpdateSheet, ProfileSelector, AddNewProfile
    releases.json   releases page and sub-components
    news.json       news page and sub-components
    webstore.json   webstore page and sub-components
    admin.json      admin panel, ban dialog, all ban strings
    common.json     404 page, shared labels
  hu/
    (same structure)
```

### Usage in components

```tsx
const { t } = useTranslation("auth");
// t("login.title") → "Login to your account" / "Bejelentkezés"
```

## Language picker locations

### Settings page (`SettingsPageContent.tsx`)

A dedicated "Language" section with 🇬🇧 English / 🇭🇺 Magyar buttons.
Active language gets a `ring-2` highlight using the current theme colours.

### Auth pages (Login, Signup, PasswordReset)

A small `<LanguageToggle />` component in the top-right corner of each auth
card — visible before login so users can switch before creating an account.
Component lives at `src/components/ui/LanguageToggle.tsx`.

## Notable implementation decisions

### Hungarian grammar

Hungarian is agglutinative — translations are written as complete context-aware
strings rather than assembled from fragments. For example ban duration labels
("1 óra", "7 nap") are full strings in the JSON, not built from number + unit.
The `bannedUntil` key uses i18next interpolation: `"Tiltva {{date}}-ig"` with
`t("detail.bannedUntil", { date: formattedDate })`.

### getRoleConfig in UserDetail

`getRoleConfig` is a plain function (not a hook), so `useTranslation` cannot be
called inside it. Fixed by passing `t` as a second parameter:
`getRoleConfig(role, t)`. The function signature is updated accordingly.

### PRESET_REASONS in BanDialog

The preset reason chips were a module-level array. Moved to a keys array
(`PRESET_REASONS_KEYS`) at module level; the translated strings are derived
inside the component via `.map((key) => t(`ban.${key}`))` so `t` is available.

### Preset duration labels in BanDialog

`BanPresetCard` previously received `preset.description` and `preset.label`
from the module-level `PRESETS` array in `useBanDialogLogic.ts`. These are now
replaced at the render site with `t(\`ban.presets.${preset.id}\`)`using the
preset's`id` (`"1h"`, `"7d"` etc.) which maps directly to the translation keys.

### Adding more languages

1. Add a new folder under `src/locales/` (e.g. `de/`) with the same 9 JSON files
2. Import all files in `src/lib/i18n.ts` and add to the `resources` object
3. Add a button in `SettingsPageContent.tsx` and `LanguageToggle.tsx`
4. Add `"de"` to the `Language` type in `SettingsContext.tsx`
